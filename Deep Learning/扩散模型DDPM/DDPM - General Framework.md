## 1. 问题定义：DDPM 在解决什么

### 1.1 核心思想

DDPM 将图像生成建模为一个**渐进式去噪**过程：

| 阶段 | 做什么 | 数学本质 |
|------|--------|---------|
| **前向过程（Forward）** | 给真实图像逐步加噪，直到变成纯噪声 | 固定的马尔可夫链，不需要学习 |
| **反向过程（Reverse）** | 从纯噪声逐步去噪，恢复出图像 | 需要学习的马尔可夫链，用神经网络预测每一步该去掉的噪声 |

前向过程是**人为定义的**（破坏过程），反向过程是**需要学习的**（修复过程）。

### 1.2 与其他生成模型的本质区别

| 模型 | 生成方式 | 特点 |
|------|---------|------|
| **GAN** | 一步到位：噪声 → 图像 | 快但训练不稳定，模式坍塌 |
| **VAE** | 一次采样 + 解码 | 快但生成质量通常偏低 |
| **DDPM** | 逐步迭代去噪（通常 1000 步） | 慢但质量高、训练稳定、模式覆盖好 |
| **Flow-based** | 可逆变换 | 精确似然但结构受限 |

DDPM 的 trade-off 很明确：**用推理速度换生成质量和训练稳定性**。

### 1.3 数学框架（最简版本）

只需要理解 3 个公式：

**(1) 前向加噪（闭式解）**：
$$x_t = \sqrt{\bar{\alpha}_t} \cdot x_0 + \sqrt{1-\bar{\alpha}_t} \cdot \epsilon, \quad \epsilon \sim \mathcal{N}(0, \mathbf{I})$$

一步就能从干净图像 $x_0$ 算出任意时间步的加噪图像 $x_t$，**无需迭代**。

**(2) 训练目标**：
$$\mathcal{L} = \left\| \epsilon_\theta(x_t, t) - \epsilon \right\|^2$$

神经网络只做一件事：**预测噪声**，然后计算 MSE。

**(3) 反向采样（逐步去噪）**：
$$x_{t-1} = \frac{1}{\sqrt{\alpha_t}}\left(x_t - \frac{\beta_t}{\sqrt{1-\bar{\alpha}_t}} \epsilon_\theta(x_t, t)\right) + \sigma_t \cdot z$$

从 $x_T$（纯噪声）出发，每一步用网络预测的噪声算出均值，加上随机扰动，迭代到 $x_0$。

> **这三个公式就是整个 DDPM 的数学核心**。任何实现都是对这三个公式的工程化。

## 2. 框架总览：六大模块

任何一个 DDPM 实现，无论用什么框架、什么语言，都逃不开这 6 个模块：

```
┌─────────────────────────────────────────────────────────────────┐
│                        DDPM 通用框架                             │
├───────────────┬─────────────────────────────────────────────────┤
│  模块一        │  噪声调度 (Noise Schedule)                      │
│               │  定义 β_t 序列，控制"加多少噪声"                  │
├───────────────┼─────────────────────────────────────────────────┤
│  模块二        │  时间编码 (Time Encoding)                       │
│               │  让网络知道"当前在第几步"                        │
├───────────────┼─────────────────────────────────────────────────┤
│  模块三        │  去噪骨干网络 (Denoising Backbone)              │
│               │  实际执行去噪的神经网络，通常是 U-Net             │
├───────────────┼─────────────────────────────────────────────────┤
│  模块四        │  条件注入机制 (Conditioning)                    │
│               │  将额外控制信号（类别、文本、图像...）注入网络     │
├───────────────┼─────────────────────────────────────────────────┤
│  模块五        │  训练管线 (Training Pipeline)                   │
│               │  组织前向加噪 → 网络预测 → 损失计算 → 参数更新    │
├───────────────┼─────────────────────────────────────────────────┤
│  模块六        │  采样管线 (Sampling Pipeline)                   │
│               │  组织反向去噪循环，从噪声生成图像                  │
└───────────────┴─────────────────────────────────────────────────┘
```

**模块间依赖关系**：

```
模块一（噪声调度）
    │
    ├──→ 模块五（训练管线）──→ 需要模块二（时间编码）+ 模块三（骨干网络）+ 模块四（条件注入）
    │
    └──→ 模块六（采样管线）──→ 同样需要模块二 + 模块三 + 模块四
```

模块一提供系数，模块二/三/四构成网络，模块五和模块六是两条使用这个网络的不同流程。

## 3. 模块一：噪声调度（Noise Schedule）

### 3.1 它是什么

噪声调度定义了 $\beta_t$ 序列（$t=1,2,...,T$），即**每一步前向过程中加入多少噪声**。

### 3.2 它决定了什么

$\beta_t$ 是整个扩散过程的**唯一超参数序列**，它决定了：

| 影响                                        | 说明                             |
| ----------------------------------------- | ------------------------------ |
| $\alpha_t = 1 - \beta_t$                  | 单步信号保留率                        |
| $\bar{\alpha}_t = \prod_{s=1}^t \alpha_s$ | 累积信号保留率（$x_t$ 中还剩多少 $x_0$ 的信息） |
| 前向加噪速度                                    | $\bar{\alpha}_t$ 下降到 0 的速度     |
| 反向去噪难度                                    | 每步要去掉多少噪声                      |
| 后验方差 $\sigma_t^2$                         | 反向采样时的随机性大小                    |

### 3.3 两种主流策略

| 策略 | 核心思想 | 特点 | 适用场景 |
|------|---------|------|---------|
| **Linear** | $\beta_t$ 线性增长 | 简单、直观；中间时间步信号保留较少 | 入门、快速实验 |
| **Cosine** | $\bar{\alpha}_t$ 按余弦曲线下降 | 噪声增长更平滑；中间步保留更多信息 | 追求更高生成质量 |

**选择原则**：
- 刚开始实现时用 Linear，确保流程跑通
- 追求质量时切换 Cosine，通常有明显提升
- 两者在 $t \to T$ 时都会趋近于纯噪声，区别在于中间过程

### 3.4 从 $\beta_t$ 推导出所有需要的系数

有了 $\beta_t$ 序列后，其余系数都是**确定性推导**，没有选择空间：

```
β_t  ──→  α_t = 1 - β_t
       ──→  ᾱ_t = cumprod(α_t)
       ──→  √ᾱ_t = sqrt(ᾱ_t)
       ──→  √(1-ᾱ_t) = sqrt(1 - ᾱ_t)
       ──→  1/√α_t = rsqrt(α_t)
       ──→  σ_t² = β_t · (1-ᾱ_{t-1}) / (1-ᾱ_t)
```

这些系数各有用处：

| 系数 | 用在哪 | 用途 |
|------|--------|------|
| $\sqrt{\bar{\alpha}_t}$ | 训练 | 前向加噪时的信号系数 |
| $\sqrt{1-\bar{\alpha}_t}$ | 训练 | 前向加噪时的噪声系数 |
| $1/\sqrt{\alpha_t}$ | 采样 | 反向均值计算 |
| $\beta_t$ | 采样 | 反向均值计算 |
| $\sqrt{1-\bar{\alpha}_t}$ | 采样 | 反向均值计算 |
| $\sigma_t^2$ | 采样 | 反向采样时的方差 |

### 3.5 关键设计决策

| 决策 | 选项 | 影响 |
|------|------|------|
| $T$ 取多大 | 通常 1000；也可 200、4000 | 越大质量越好但推理越慢 |
| $\beta_1$ 起始值 | 通常 $10^{-4}$ | 第一步噪声不能太大，否则直接破坏图像 |
| $\beta_T$ 终止值 | 通常 0.02（linear）或由公式推（cosine） | 要保证 $\bar{\alpha}_T \approx 0$（即 $x_T$ 接近纯噪声） |
| 是否学习 $\beta_t$ | 固定 / 可学习 | 绝大多数实现用固定值 |

## 4. 模块二：时间编码（Time Encoding）

### 4.1 为什么需要

神经网络需要知道当前处理的是**第几步**——$t=1$（噪声很少，图像几乎清晰）和 $t=999$（几乎纯噪声）需要完全不同的去噪策略。时间编码就是把这个**离散的步数 $t$** 变成一个**连续的向量表示**，注入到网络中。

### 4.2 设计空间

时间编码有两个基本问题：

**(1) 用什么编码方式？**

| 方式 | 原理 | 特点 |
|------|------|------|
| **正弦位置编码** | 用不同频率的 sin/cos 函数 | Transformer 经典方法，参数无关，可泛化到训练时未见过的 $t$ |
| **可学习嵌入** | 维护一个 `Embedding(T, dim)` 查表 | 参数更多但需要学习，不能泛化到未见过的 $t$ |
| **随机傅里叶特征** | 用随机高斯矩阵投影 | 近似核方法，Nerf 中使用较多 |

绝大多数 DDPM 用**正弦位置编码**——简单有效且零参数。

**(2) 编码之后如何处理？**

```
t (整数) → 编码向量 (dim 维) → [可选: MLP 非线性变换] → 最终的 t_emb
```

大多数实现会在正弦编码后加一两个 MLP 层做非线性变换，让时间表示更丰富。

### 4.3 正弦位置编码的数学直觉

核心公式：
$$\text{PE}(t, 2i) = \sin(t \cdot \omega_i), \quad \text{PE}(t, 2i+1) = \cos(t \cdot \omega_i)$$

其中频率 $\omega_i$ 从 $1$ 指数递减到 $1/10000$。

**直觉理解**：
- 低频分量 → 捕捉"早期还是晚期"的粗粒度信息
- 高频分量 → 捕捉"精确是第几步"的细粒度信息
- 多频率叠加 → 网络可以同时从不同尺度理解时间位置

### 4.4 关键设计决策

| 决策 | 选项 | 影响 |
|------|------|------|
| 编码维度 | 通常 128~512 | 维度越大表示能力越强，计算量增加可忽略 |
| 频率范围 | 通常 1 ~ 1/10000 | 覆盖几个数量级，确保低频到高频都有 |
| 是否加 MLP | 加 / 不加 | 加了表示更灵活；不加也行 |
| 偶奇排列 | sin 偶数位 / cos 奇数位 | 惯例，也可以反过来 |
| 编码粒度 | 对整数 t 编码 / 对连续 t 编码 | 正弦编码天然支持连续值 |

## 5. 模块三：去噪骨干网络（Denoising Backbone）

### 5.1 它做什么

接收 $(x_t, t, \text{[可选条件]})$ 作为输入，输出**预测的噪声** $\epsilon_\theta$（与 $x_t$ 同形状）。

### 5.2 为什么是 U-Net

| 需求 | U-Net 如何满足 |
|------|---------------|
| 输入输出同尺寸 | U-Net 的 Encoder-Decoder 天然对称 |
| 需要多尺度信息 | 下采样 → 处理 → 上采样 提供多分辨率视野 |
| 需要保留细节 | Skip Connection 将浅层细节直通深层 |
| 需要条件注入 | 在每个分辨率层级都可以注入时间和条件 |

**U-Net 几乎是 DDPM 的事实标准**，但理论上任何 Encoder-Decoder 架构都可以。

### 5.3 U-Net 的通用结构

```
输入 x_t (与目标图像同尺寸)
    │
    ▼
┌─────────────────────────────────────────┐
│  Encoder（编码器 / 下采样路径）           │
│                                          │
│  每个stage:                              │
│    [ResBlock × N] → [DownSample]         │
│    分辨率减半，通道数翻倍（通常）          │
│    保存 skip 特征给解码器                 │
│                                          │
│  典型深度: 3~5 个 stage                  │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  Bottleneck（瓶颈层）                     │
│                                          │
│  最低分辨率处处理                          │
│  通常就是 1~2 个 ResBlock                 │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  Decoder（解码器 / 上采样路径）           │
│                                          │
│  每个stage:                              │
│    [UpSample] → concat(skip特征) → [ResBlock × N]  │
│    分辨率翻倍，通道数减半（通常）          │
│                                          │
│  与 Encoder 对称                          │
└─────────────────────────────────────────┘
    │
    ▼
输出 εθ (与输入 x_t 同形状)
```

### 5.4 各组件的作用

| 组件 | 作用 | 为什么不用替代方案 |
|------|------|-------------------|
| **ResBlock** | 基本计算单元，提取特征 + 注入条件 | 残差连接保证梯度流动，深层网络也能训练 |
| **DownSample** | 降低分辨率增大感受野 | 可选：stride=2 卷积、MaxPool、AvgPool |
| **UpSample** | 恢复分辨率 | 最近邻+卷积（避免棋盘伪影）vs TransposedConv |
| **Skip Connection** | 浅层特征直达深层 | 没有的话细节丢失严重，生成模糊 |
| **GroupNorm** | 归一化 | 比 BatchNorm 更适合小 batch 场景 |
| **SiLU/Swish** | 激活函数 | 扩散模型中效果优于 ReLU |
| **Attention** | 可选：在低分辨率层加入自注意力 | 增强全局一致性，做大图像时通常需要 |

### 5.5 网络规模的设计空间

| 维度 | 小网络 | 大网络 | 本项目选择 |
|------|--------|--------|-----------|
| 基础通道数 | 32~64 | 128~256 | 64 |
| Encoder 深度 | 3 层 | 5~6 层 | 3 层 |
| 通道倍增策略 | 每层 ×2 | 每层 ×2 或更激进 | 64→128→256 |
| 每层 ResBlock 数 | 1 | 2~3 | 1 |
| Attention 层 | 无 / 只在最深 | 多个分辨率层 | 无（轻量设计） |
| 总参数量 | 几百万 | 几千万到几亿 | ~几百万 |

### 5.6 关键设计决策

| 决策 | 考量 |
|------|------|
| 网络大小 | 与数据集复杂度匹配。简单数据集用小网络即可，复杂自然图像需要大网络 + Attention |
| Encoder 深度 | 决定了最小分辨率。$128 \to 64 \to 32$（3层）通常够用 |
| 通道倍增倍数 | 常规是 2×，也可以 1.5× 或 3× |
| 是否用 Attention | 要做高分辨率自然图像？加。简单的灰度图？不必。 |
| Normalization 类型 | 小 batch → GroupNorm；大 batch → BatchNorm 也可以 |
| Skip 拼接方式 | concat（信息保留多）vs add（省参数），concat 为主流 |
| 预测目标 | 预测噪声 $\epsilon$（主流） / 预测 $x_0$ / 预测 $v$（速度预测） |

## 6. 模块四：条件注入机制（Conditioning）

### 6.1 为什么需要条件

无条件 DDPM 只能从噪声中随机生成图像——你无法控制它生成什么。条件注入让你能告诉网络："我要生成一只猫"、"我要从这个文字描述生成"、"我要从这张草稿生成精细图像"。

### 6.2 条件的类型

| 条件类型 | 例子 | 编码方式 |
|---------|------|---------|
| **离散类别** | ImageNet 类别、MNIST 数字 | One-hot 或 Embedding |
| **连续标量** | 角度、温度、强度 | 傅里叶编码（循环变量）、直接归一化 |
| **文本** | 自然语言描述 | CLIP / T5 等预训练文本编码器 |
| **图像** | 草图、分割图、低分辨率图 | CNN/UNet Encoder 或直接 concat |
| **多模态组合** | 类别 + 角度 + 文本 | 各自编码后拼接或加法融合 |

### 6.3 条件如何注入网络

有三种基本模式：

```
模式1: 加法偏置 (Additive Bias)
─────────────────────────────────
条件 → Embedding/MLP → 向量 → reshape → 加到特征图上

特点: 轻量、每个 ResBlock 都注入
举例: 本项目、大多数类别条件 DDPM
适用: 简单的全局条件

模式2: 通道拼接 (Channel Concatenation)
─────────────────────────────────
条件(如图像) → 编码 → 与 x_t 在通道维拼接 → 一起送入 UNet

特点: 信息量更大、只注入一次
举例: 图像超分辨率、图像修复
适用: 条件本身就是图像或空间信息

模式3: 交叉注意力 (Cross-Attention)
─────────────────────────────────
条件 → 编码为序列 → UNet 中间层用 Cross-Attention 查询

特点: 最灵活、计算量最大
举例: Stable Diffusion 等文本到图像模型
适用: 文本描述、长序列等复杂条件
```

**本项目用的是模式1（加法偏置）**——类别 one-hot 和方位角向量拼接后，通过一个小 MLP 映射为逐通道偏置，在**每个 ResBlock** 中加到特征图上。

### 6.4 连续循环变量的特殊处理

当条件是**角度、方向、相位等循环变量**时（0° = 360°），不能直接输入数值：

| 问题 | 错误做法 | 正确做法 |
|------|---------|---------|
| 359° 和 1° 实际上很近 | 直接输 359 vs 1 → 网络认为差很大 | 用 $(\cos\theta, \sin\theta)$ 编码 → 网络理解它们是近似向量 |
| 高阶变化模式 | 只有一阶 → 缺少细节 | 多阶 $(\cos k\theta, \sin k\theta), k=1,2,...,K$ |

**通用原则**：任何循环变量都需要用三角编码，阶数越高表示能力越强。

### 6.5 关键设计决策

| 决策 | 考量 |
|------|------|
| 注入方式 | 简单条件用加法偏置，图像条件用通道拼接，文本条件用 Cross-Attention |
| 注入位置 | 只注入一次（输入层）vs 每层注入（信息贯穿整个网络） |
| 条件编码维度 | 与条件复杂度匹配，简单条件 64~128 足够 |
| 条件融合方式 | 加法（省参数）vs 拼接（信息量大）vs FiLM（逐通道 scale+shift） |
| 无分类器引导（CFG） | 训练时随机丢掉条件，推理时用有/无条件输出的差值引导 |

## 7. 模块五：训练管线（Training Pipeline）

### 7.1 训练流程（抽象）

```
for each iteration:
    ┌─────────────────────────────────────────────┐
    │ 1. 取一批真实图像 x_0                         
    │ 2. 取对应的条件 c（可选）                      
    │ 3. 随机采样时间步 t ~ Uniform(0, T)           
    │ 4. 随机采样噪声 ε ~ N(0, I)                   
    │ 5. 计算 x_t = √ᾱ_t·x_0 + √(1-ᾱ_t)·ε          
    │ 6. 网络预测: εθ = model(x_t, t, c)           
    │ 7. 计算损失: loss = ||εθ - ε||²              
    │ 8. 反向传播，更新参数                         
    └─────────────────────────────────────────────┘
```

### 7.2 为什么这样训练

DDPM 原始论文推导了一个复杂的变分下界（ELBO）作为训练目标。但作者发现：

**简化为只预测噪声 + MSE 损失，效果一样好，实现极简单**。

这是 DDPM 的一个重要贡献——把看似复杂的生成模型训练简化成了一个回归问题。

### 7.3 训练技巧

| 技巧                 | 说明                                                  | 重要程度   |
| ------------------ | --------------------------------------------------- | ------ |
| **均匀采样 t**         | 每个 iteration 从 1~T 均匀随机采样，确保所有时间步都被充分训练             | 必须     |
| **数据归一化到 [-1, 1]** | 因为 $x_T \sim \mathcal{N}(0,\mathbf{I})$，输入输出都是零均值分布 | 必须     |
| **EMA（指数移动平均）**    | 对模型参数做平滑，推理时用 EMA 参数通常质量更好                          | 推荐     |
| **Dropout 条件**     | 训练时随机将条件置零，推理时可做无分类器引导（CFG）                         | 条件模型推荐 |
| **梯度裁剪**           | 防止单步梯度爆炸                                            | 可选     |
| **混合精度训练**         | FP16 加速，节省显存                                        | 大模型推荐  |
| **数据增强**           | 水平翻转等简单增强，注意条件也要同步变换                                | 小数据集推荐 |

- **有关数据归一化到 [-1, 1]：**
	- 归一化数据是训练图像。也就是从数据集加载的真实 SAR 图像（PNG 文件），需要从 `[0, 1]` 缩放到 `[-1, 1]`。
	- 为什么必须做这个缩放？
		- DDPM 的核心假设是：
		  $$x_T \sim \mathcal{N}(0, \mathbf{I})$$
		  即扩散到最后一步时，图像变成了**均值为 0、方差为 1** 的纯噪声。
		  那倒推回来——训练时每一步的 $x_t$ 是什么？
		  $$x_t = \sqrt{\bar{\alpha}_t} \cdot x_0 + \sqrt{1-\bar{\alpha}_t} \cdot \epsilon$$
		  它是 $x_0$ 和纯噪声的加权和。两边要"兼容"：
			- 如果 $x_0 \in [0, 1]$（均值 0.5）：
			  $x_t = \text{信号(均值0.5)} + \text{噪声(均值0)}$ → 分布不一致，均值在变，网络要同时适应“均值漂移”、“去噪”两件事，网络额外学习适应这个漂移，训练变得不稳定，生产质量下降。
			- 如果 $x_0 \in [-1, 1]$（均值 0）：
			  $x_t = \text{信号(均值0)} + \text{噪声(均值0)}$ → 分布一致，均值始终为 0，网络专注“去噪”。
	
	- 生成完之后还要还原回缩放前的样子去。

	- 为什么真实 SAR 图像数据是 `[0, 1]`：原始 PNG 的像素值是 0~255 的整数，人为==除以了 255== 做的预处理，这是深度学习处理图像的通用惯例。
	- 为什么不直接保留 0~255？
		- 用 0~255：数值大，梯度也大，容易爆炸。和网络权重（通常 ±0.02 左右初始化）不匹配。
		- 用 0~1：数值小，梯度稳定。和网络权重在同一数量级。几乎是所有图像任务的默认做法。



### 7.4 损失函数变体

| 变体 | 公式 | 特点 |
|------|------|------|
| **简单 MSE（本项目）** | $\|\epsilon_\theta - \epsilon\|^2$ | 标准做法 |
| **加权的 MSE** | $w(t) \cdot \|\epsilon_\theta - \epsilon\|^2$ | 给某些时间步更高权重 |
| **预测 $x_0$ 而非 $\epsilon$** | $\|x_0^\theta - x_0\|^2$ | 等价但需要更多计算 |
| **预测 $v$ (velocity)** | $\|v_\theta - (\sqrt{\bar{\alpha}_t}\epsilon - \sqrt{1-\bar{\alpha}_t}x_0)\|^2$ | Improved DDPM 提出，SNR 加权 |

### 7.5 关键设计决策

| 决策 | 考量 |
|------|------|
| 预测目标 | $\epsilon$ 是最主流的选择 |
| 是否用 EMA | 用。几乎零成本提升质量 |
| Batch Size | 受显存限制。越小 → GroupNorm 越重要 |
| $t$ 的采样分布 | 均匀采样为主流；也可以按重要性采样 |
| 条件丢弃概率 | CFG 时通常 10~20% 丢弃 |

## 8. 模块六：采样管线（Sampling Pipeline）

### 8.1 采样流程（DDPM Algorithm 2）

```
输入: 训练好的模型, 目标条件 c（可选）, 噪声调度系数
输出: 生成的图像 x_0

1. 从标准正态分布采样 x_T ~ N(0, I)

2. for t = T-1, ..., 0:          (逐步去噪)
     ┌─────────────────────────────────────────────┐
     │ a. 网络预测噪声: εθ = model(x_t, t, c)        │
     │ b. 计算反向均值:                              │
     │    μ = (1/√α_t) · (x_t - β_t/√(1-ᾱ_t) · εθ)  │
     │ c. 如果 t > 0:                                 │
     │      x_{t-1} = μ + σ_t · z    (z ~ N(0,I))     │
     │    否则 (t = 0):                               │
     │      x_0 = μ                    (确定性输出)    │
     └─────────────────────────────────────────────┘

3. 返回 x_0
```

### 8.2 为什么 t=0 时不加噪声

最后一步如果还加随机噪声，图像会带一层模糊。去噪的最后一步应该**确定性**地给出最清晰的图像。

### 8.3 加速采样

DDPM 最大的痛点是**推理太慢**（T 步迭代）。已有多种方法减少步数：

| 方法 | 核心思想 | 步数 | 质量 |
|------|---------|------|------|
| **DDPM（本项目）** | 完整 T 步采样 | 1000 | 最高 |
| **DDIM** | 非马尔可夫跳步采样 | 50~100 | 接近 DDPM |
| **DDIM respacing** | 均匀间隔取子序列 | 10~100 | 可接受 |
| **DPM-Solver** | ODE 数值求解器 | 10~20 | 很好 |
| **LCM（潜在一致性模型）** | 蒸馏 + 一步预测多步 | 1~4 | 较好 |

**原则**: 如果只需偶尔生成，1000 步的 DDPM 也可以；如果需要快速生成，用 DDIM 50~100 步是很好的折中。

### 8.4 条件引导采样

有条件模型时，推理阶段可以用==**引导**==来增强条件的影响：

**无分类器引导（Classifier-Free Guidance, CFG）**：

```
εθ_cfg = εθ(x_t, t, ∅) + w · (εθ(x_t, t, c) - εθ(x_t, t, ∅))
                         ↑
                    引导强度 w ≥ 1

  w=1: 标准条件生成
  w>1: 条件影响被放大（更贴合条件，但可能牺牲多样性）
  w=0: 无条件生成
```

- 需要训练时使用了条件丢弃（见 §7.3）
- `w` 越大，生成结果越"听话"但越可能过度饱和/失真
- 典型取值为 1.5~7.5

### 8.5 关键设计决策

| 决策 | 考量 |
|------|------|
| 采样算法 | DDPM（完整）/ DDIM（加速）/ DPM-Solver（高速） |
| T 的大小 | 训练用多少，推理就可以用多少；也可以小于训练时的 T |
| 是否用 CFG | 条件模型强烈推荐，大幅提升条件一致性 |
| 引导强度 w | 根据应用调整：高保真任务用大 w，多样性任务用小 w |
| 随机种子控制 | 相同噪声 x_T + 相同条件 → 相同结果（可复现） |

## 9. 条件式 DDPM 的扩展维度

从本项目出发，条件 DDPM 可以从以下维度扩展：

```
                    条件类型
                       │
    离散类别 ──────────┼────────── 连续标量
    (one-hot/embed)    │         (归一化/傅里叶编码)
                       │
        文本 ──────────┼────────── 图像
    (Cross-Attention)  │         (通道拼接/Encoder)
                       │
        多条件融合 ────┴──── 无分类器引导
                       │
                       ▼
                   扩展的条件 DDPM
```

### 具体扩展方向

| 从本项目 | 扩展到 | 需要改动 |
|---------|--------|---------|
| 10维 one-hot 类别 | 文本描述（CLIP embedding） | 条件编码改为 Cross-Attention |
| 10维方位角编码 | 任意角度范围的连续条件 | 调整三角编码阶数 |
| 单通道灰度图 | RGB 彩色图 | 改 `img_channels=3` |
| 128×128 分辨率 | 256×256 或更高 | 加深 U-Net，加 Attention |
| 简单加法偏置 | FiLM（scale + shift） | 每个 ResBlock 加入两个 MLP |
| 无引导 | Classifier-Free Guidance | 训练时随机丢条件，推理时差值引导 |
| DDPM 1000步 | DDIM 50步 | 改采样循环，支持跳步 |
| 单一条件 | 多条件组合 | 各条件编码后 append 再融合 |

## 10. 设计决策清单

下面是在实现自己的 DDPM 时需要依次做出的关键决策：

### 10.1 数据层面

- [ ] 图像分辨率：多大？是否需要预处理/裁剪？
- [ ] 通道数：灰度（1）还是彩色（3 或 4）？
- [ ] 数据归一化范围：`[0,1]` → 训练时映射到 `[-1,1]`（标准做法）
- [ ] 数据增强：是否需要？条件如何同步变换？

### 10.2 扩散过程

- [ ] 总步数 T：1000（标准）、100（快速）、还是 4000（高质量）？
- [ ] 噪声调度策略：Linear 还是 Cosine？
- [ ] $\beta_1$ 和 $\beta_T$ 的取值
- [ ] 学习 $\beta_t$（可学习方差）还是固定？

### 10.3 网络架构

- [ ] 基础通道数：规模与数据集匹配
- [ ] Encoder/Decoder 深度：决定了多尺度的层级数
- [ ] 每层 ResBlock 数量
- [ ] 是否加入 Attention 层？在哪些分辨率加？
- [ ] Normalization：GroupNorm 的组数
- [ ] 预测目标：$\epsilon$ / $x_0$ / $v$

### 10.4 条件机制

- [ ] 条件类型和数量
- [ ] 每种条件的编码方式：one-hot / embedding / CNN / CLIP / 傅里叶
- [ ] 注入方式：加法偏置 / 通道拼接 / Cross-Attention / FiLM
- [ ] 注入位置：仅在输入层 / 每个 ResBlock / 特定层
- [ ] 是否使用无分类器引导？

### 10.5 训练

- [ ] 优化器：Adam/AdamW，学习率，权重衰减
- [ ] Batch Size
- [ ] 训练轮数
- [ ] EMA 衰减率
- [ ] 条件丢弃概率（如需 CFG）
- [ ] 混合精度训练
- [ ] 梯度裁剪阈值

### 10.6 推理

- [ ] 采样算法：DDPM / DDIM / DPM-Solver
- [ ] 推理步数：与训练一致还是更少
- [ ] CFG 引导强度 w
- [ ] 批量生成 vs 单张生成
