用真实图片的条件，让模型重新生成一遍，然后对比验证生成质量。

这是 ==**DDPM 评估阶段**==的标准做法：
```
真实图片的条件
   (label=3, az=45.0°)
         │
         ▼
   DDPM 模型生成     ────→   生成图 (label=3, az=45.0°)
                             真实图 (label=3, az=45.0°)  ← 原始图片
         │
         ▼
   对比：生成图 vs 真实图
   （FID、IS、人工视觉评估等）
```

## 具体到这个项目，猜测三个目的
1. 质量评估
生成图和真实图条件一致（同类别 + 同角度），可以公平地评估生成质量。比如对比 "class 3、45°" 的生成图和真实图，看 DDPM 学没学到 SAR 目标的散射特征。

2. 数据增强
如果生成质量过关，这些合成 SAR 图可以喂给下游分类器做训练数据补充。SAR 真实数据获取成本高（需要雷达飞行采集），用 DDPM 扩充数据集是常见做法。

3. 条件控制验证
通过和真实图的直接对照，验证条件机制是否生效——比如输入 `class=3, az=90°`，生成的图是否真的有 class 3 的特征、阴影方向是否对应 90° 照射。


## 用同一个 txt 的原因很简单

```
不是：为了"还原"那些真实图片
而是：为了在相同条件下，生成新的图来"对比"真实图片
```
txt 文件就是提供了一套标准的测试条件集，确保评估可复现。如果每次随机条件，就没法稳定对比生成质量了。
