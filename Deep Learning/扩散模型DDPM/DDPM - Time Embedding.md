# 原理解释
在 DDPM 的去噪过程中，模型需要面对 1000 个不同模糊程度的阶段。面对一张充满噪点的图，U-Net 必须知道现在是第几步（$t$），才能知道当前该采用"大刀阔斧去噪"还是"精雕细琢微调"的策略。

## 1. 痛点：为什么不能直接输入数字 $t$ ？

如果你直接把数字 $t=50$ 作为一个标量扔进拥有几十万个浮点数的神经网络里，这个单薄的数字所产生的数值波动，瞬间就会被庞大的图像特征淹没，网络根本"听不见"。

## 2. 第一步：正弦位置编码（Sinusoidal Positional Embeddings）

在 DDPM 中，时间步嵌入直接借用了 Transformer 中的**正弦位置编码**技术。它的核心目的，就是把一个单薄的标量数字 $t$（比如 500），拉伸成一个拥有丰富频率信息的、长度为 $d$ 的高维向量。

**操作原理**：它把数字 $t$ 作为一个变量，代入到几十个甚至上百个不同频率的 $\sin$ 和 $\cos$ 三角函数公式中：

### 2.1 核心公式

假设我们要把时间步 $t$ 变成一个长度为 $d$（比如 128 维或 256 维）的向量。这个向量里的每一个元素（我们用索引 $i$ 来表示，范围是 $0$ 到 $d/2 - 1$），都是通过下面这两个三角函数公式计算出来的：

$$PE_{(t, 2i)} = \sin\left(\frac{t}{10000^{2i/d}}\right)$$

$$PE_{(t, 2i+1)} = \cos\left(\frac{t}{10000^{2i/d}}\right)$$

- $t$：当前的时间步（例如 1 到 1000 之间的整数）。
- $d$：**目标特征向量的总维度**（例如 128 维）。
- $i$：向量中**特定维度的索引**。偶数维度（$2i$）用 $\sin$ 算，奇数维度（$2i+1$）用 $\cos$ 算。
- $10000^{2i/d}$：这是一个频率控制项。随着维度索引 $i$ 越来越大，分母会呈指数级增大，导致整个 $\sin$ 或 $\cos$ 函数的波长变长（频率变慢）。

- **物理效果**：原本单薄的数字"500"，被拉长成了一个包含丰富频率信息的"高维条形码"（例如一个 256 维的向量）。
- **为什么非用三角函数？**：因为三角函数具有极佳的连续性和周期性。它能让网络在数学上极其平滑地理解"$t=50$ 和 $t=51$ 是紧挨着的兄弟"，同时也能轻易区分"早期"和"晚期"的宏观频率差异。

### 2.2 这个公式的物理意义是什么？

如果你盯着公式看，可能会觉得抽象，但它的物理画面极其优美：

- **低维度（$i$ 很小）：** 分母接近 1，$\sin(t)$ 的变化极快，就像秒针。哪怕 $t$ 只从 500 变成 501，这部分的值也会发生剧烈变化。
- **高维度（$i$ 很大）：** 分母变得极大（接近 10000），波浪的变化极度缓慢，就像时针。它可能几十个时间步才会产生微小的起伏。

### 2.3 这样设计的绝妙之处在于：

1. **绝对位置表达**：每个时间步 $t$ 生成的高维向量都是绝对唯一的，就像一个由无数个不同频率的波叠加而成的"高维条形码"。
2. **相对距离感知**：因为三角函数的连续性，网络可以通过简单的线性变换，轻易算出 $t=50$ 和 $t=51$ 在"时间"上是相对接近的。

## 3. 第二步：在 U-Net 内部"全网广播"（Add 操作）

### 3.1 编码之后的第一步：MLP 映射

刚刚我们算出来的正弦/余弦向量并不会直接"裸奔"进入 U-Net。

- **操作**：它首先会被送入一个**多层感知机（MLP）**（通常是两个全连接层，中间夹着一个 SiLU 激活函数）。

在深度学习的文献和教程中，**多层感知机（MLP）**、**全连接层（Fully Connected Layer）** 和 PyTorch 里的 **线性层（Linear Layer）**，其实在大多数语境下指代的都是同一个东西的组成部分。

在 PyTorch 中，线性层的代码名叫 `nn.Linear(in_features, out_features)`。它的作用是接收一组输入的数字（特征向量），然后通过内部无数根"导线"（权重 Weights）将它们互相混合，最后输出一组新的数字。数学本质：$y = Wx + b$。


> [!note] 
> DDPM 在这里搭建了一个小型的 MLP（多层感知机）加工厂，流程如下：
>
> 1. **第一个线性层 (Linear 1)**：
>     - 它接收原始的 128 维正弦波向量，然后通过内部的"旋钮"（权重），通常会把它**升维 / 膨胀**（比如放大到 512 维）。这就好比把原始特征打碎、展开，给网络更大的"思考空间"。
>
> 2. **SiLU 激活函数**：
>     - 如果只有线性层，网络就只能做直来直去的"线性变换"（画直线）。
>     - 引入 SiLU（一种比 ReLU 更平滑的激活函数）是为了加入"非线性"。这相当于给这团数据揉进了一些"曲线"，让它能够表达极其复杂的、扭曲的时间关系（比如"前 100 步变化剧烈，后 900 步变化微弱"的非线性逻辑）。
>
> 3. **第二个线性层 (Linear 2)**：
>     - 在 SiLU 把数据揉软之后，第二个线性层再次对数据进行混合，把它精准地映射到 U-Net 能够接收的标准"厚度"（即 `time_embed_dim`）。
>
> **总结：** 原始正弦编码只提供了绝对客观的"时间刻度"，而这套 "Linear + SiLU + Linear" 的组合，把它变成了一种**"可被 AI 学习、理解并动态调整的高级指令"**。
>
> - **目的**：纯数学公式算出来的三角函数是死板的。经过 MLP 后，网络可以在训练过程中"学习"并微调这个时间向量，把它变成最适合当前去噪任务的特征表达。

### 3.2 深入 U-Net 残差块：三大融合操作

当这个经过 MLP 打磨好的 1D 时间向量，来到 U-Net 内部的一个残差块（ResBlock）门前时，真正的魔术就开始了。它会经历以下极其严格的三个步骤：

#### 第一步：对齐通道

时间向量每次进入一个**不同的层级**，其通道数（厚度）是完全不一样的。

- **操作**：时间向量会再次经过一个简单的**全连接层**，强行将其维度伸缩到与**当前 U-Net 层级**的图像通道数完全一致。
    - **效果**：如果当前层的图像特征图厚度是 256 个通道，那么时间向量就被精确映射为 256 维。这时候，时间向量和图像特征在"厚度"上对齐了。

#### 第二步：空间广播 (Spatial Broadcasting)

- **痛点**：此时的图像特征是 3D 的（比如 `256通道 x 32高 x 32宽`），而时间向量只有 1D（`256`），形状不匹配，在数学上无法直接相加。
- **操作**：系统会在空间维度（长和宽）上，把这个 1D 的时间向量疯狂复制、拉伸。

#### 第三步：相加注入 (Addition)：纯粹的数值相加

- **操作**：系统将刚刚广播拉伸出来的"三维时间积木"，与主干路上的"三维图像特征图"进行**逐元素相加（Add）**。
    - **效果**：时间信息就像盐溶于水一样，均匀且深深地渗透到了这层图像特征的每一个像素点里。当前层的所有卷积核在这一刻，都接收到了强烈的"当前是第 $t$ 步"的指挥信号。

# 代码实现：标准正弦时间编码

DDPM中使用正弦位置编码将时间步 $t$ 映射为高维嵌入向量，公式为：

$$
\text{PE}(t, 2i) = \sin\left(t \cdot \omega_i\right), \quad
\text{PE}(t, 2i+1) = \cos\left(t \cdot \omega_i\right)
$$

其中 $\omega_i = \frac{1}{10000^{2i/d}}$。


```python
import math
import torch
import torch.nn as nn
import torch.nn.functional as F


class SinusoidalPosEmb(nn.Module):
    """标准正弦时间步嵌入。

    将时间步 t（标量或一维张量）映射为 dim 维嵌入向量。
    原理同 Transformer 位置编码，用多频率正余弦波表示时间信息。
    """

    def __init__(self, dim: int):
        super().__init__()
        self.dim = dim

    def forward(self, t: torch.Tensor) -> torch.Tensor:
        """
        Args:
            t: (B,) int64 or float — 每个样本的时间步
        Returns:
            (B, dim) — 正弦时间步嵌入向量
        """
        half_dim = self.dim // 2  # 向下取整除

        # 频率衰减底数（对数形式）
        # 等价于 exp(-log(10000) * arange(half_dim) / (half_dim - 1))
        emb = math.log(10000) / (half_dim - 1)

        # 生成一维频率向量 (half_dim,)，值从 1 指数递减至 ~1/10000
        emb = torch.exp(torch.arange(half_dim, device=t.device) * -emb)

        if t.dim() == 1:    # 防御性写法
            t = t[:, None]  # 变成列向量便于广播乘法

        # 外积 (B, half_dim)：每个时间步 t × 固定频率向量 → 相位值
        emb = t.float() * emb[None, :]   # emb转成行向量

        # -1表示按列拼接 → (B, dim)
        emb = torch.cat([torch.sin(emb), torch.cos(emb)], dim=-1)

        # 奇数维度时末尾补零对齐
        if self.dim % 2 == 1:
            emb = F.pad(emb, (0, 1))

        return emb
```

## 流程一览

| 步骤       | 操作                                     | 输出形状            |
| -------- | -------------------------------------- | --------------- |
| 1. 频率计算  | `exp(-log(10000) * arange / (half-1))` | `(half_dim,)`   |
| 2. 时间扩展  | `t[:, None]`                           | `(B, 1)`        |
| 3. 相位计算  | `t * emb`（广播外积）                        | `(B, half_dim)` |
| 4. 正余弦映射 | `[sin, cos]` 拼接                        | `(B, dim)`      |

## 关键点

- **多频率编码**：低频分量变化慢（全局时间感），高频分量变化快（精细分辨），组合后模型能同时感知粗粒度和细粒度的时间信息。
- **连续性**：正余弦函数天然光滑，相邻时间步的嵌入向量相近，帮助模型学习时间上的平滑过渡。
- **相对关系**：正余弦编码支持通过线性变换表达时间步之间的相对偏移（类似 Transformer 位置编码的性质）。
- **奇偶对齐**：当目标维度为奇数时，末尾补零确保输出形状一致。


# 代码实现：时间步嵌入
DDPM 条件 U-Net 的经典设计模式：**顶层统一编码 + 逐层投影调制**。
## U-Net级
将"原始输入"编码成统一的嵌入向量。`t_emb`(256维) 和 `cond_emb`(64维) 会被传递给**所有** ResBlock 共享使用
```python
class ConditionalUNet(nn.Module):
	
	def __init__(self, img_channels: int = 1, base_ch: int = 64, time_dim: int = 256, cond_dim: int = 64):
		......
		
		# 将原始输入编码为统一的嵌入向量
		self.time_mlp = nn.Sequential(
		    SinusoidalPosEmb(time_dim),      # 标量 t → 正弦位置编码 (256维)
		    nn.Linear(time_dim, time_dim * 2),
		    nn.SiLU(),
		    nn.Linear(time_dim * 2, time_dim),  # → t_emb (256维)
		)
		self.cond_mlp = nn.Sequential(
		    nn.Linear(20, cond_dim),   # [class_onehot(10)+azimuth(10)] = 20维
		    nn.SiLU(),
		    nn.Linear(cond_dim, cond_dim),    # → cond_emb (64维)
		)
		
		......
```

## ResBlock级
将全局嵌入向量**投影到当前 ResBlock 的输出通道数**，然后以 `(B, C, 1, 1)` 的形式空间广播加到特征图上
```python
class ResBlock(nn.Module):
	
	def __init__(self, in_ch: int, out_ch: int, time_dim: int, cond_dim: int):
		......
		
		# 将统一嵌入投影到当前block的通道数
		self.time_mlp = nn.Sequential(
		    nn.SiLU(),
		    nn.Linear(time_dim, out_ch),   # 256 → out_ch (64/128/256)
		)
		self.cond_mlp = nn.Sequential(
		    nn.SiLU(),
		    nn.Linear(cond_dim, out_ch),   # 64 → out_ch (64/128/256)
		)
		
		......
```
